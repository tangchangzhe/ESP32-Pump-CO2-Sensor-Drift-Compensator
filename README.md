# ESP32 闭路气路 CO2 监测系统（气泵式，带漂移补偿）

[English Documentation](README_EN.md)

基于 ESP32 的闭路气路 CO2 监测系统（气泵式），适用于密封腔体（如微生物培养瓶），内置多锚点漂移补偿算法弥补低成本装置气密性不足导致的数据漂移。

## 应用场景

本项目最初为监测密封腔体内藻类的呼吸与光合作用而开发，可以给以下场景提供思路参考：

- 小型密封腔体内生物（藻类、微生物等）的气体代谢监测
- 气路密封性较差的低成本实验装置
- 需要长期连续监测 CO2 或其他各种类型传感数据的场景
- 传感器存在长期漂移问题的应用

## 核心特性

### 多锚点漂移补偿算法

**问题**：密封气路不可避免会随时间泄漏，导致 CO2 读数出现系统性下降趋势，掩盖真实的生物活动信号。

**背景**：以藻类培养为例，在光暗周期中：
- **黑暗期**：藻类仅进行呼吸作用，释放 CO2，浓度应**上升**
- **光照期**：光合作用强于呼吸作用，净吸收 CO2，浓度应**下降**
- **完整周期后**：由于光合作用的净效应，CO2 浓度通常会有所降低（而非回归原点）

然而，由于气路漏气，整体数据会叠加一个持续的下降趋势。这导致：
- 黑暗期本应上升的曲线变成缓慢下降（或上升幅度被削弱）
- 光照期的下降幅度被放大

**解决方案**：前端实现的多锚点漂移补偿算法。由于漏气导致的下降趋势是非线性的（近似指数衰减），漂移速率随时间变化，因此需要多个锚点来捕捉不同时段的漂移速率：

1. **锚点选择（周期法）**：选取完整光暗周期的起止点作为锚点。由于漂移速率本身无法直接观测，而一个完整周期内既包含上升段（黑暗期）也包含下降段（光照期），对整个周期计算的斜率可以近似为该时段漂移速率的"中值"——既能将黑暗期被压低的曲线抬升回正常的上升趋势，又不会过度补偿导致光照期的下降曲线错误地向上
   > *注意：此方法为经验性估算，仅适用于漂移趋势明显可辨的场景，用于还原大致的周期性波动图像，不建议用于精密定量分析

2. **斜率计算**：对每个锚点时段进行线性回归，提取该时段的斜率作为该时段的漂移速率
3. **插值过渡**：锚点之间线性插值，实现漂移速率在不同时段间平滑过渡
4. **累积修正**：对漂移速率积分，从原始读数中减去累积漂移量

**数学表达**：

- 锚点漂移速率（线性回归斜率）：
 
 $$slope = \frac{n \sum x_i y_i - \sum x_i \sum y_i}{n \sum x_i^2 - (\sum x_i)^2}$$

- 修正值（累积积分）：
 
 $$CO_{2,corrected} = CO_{2,raw} - \int_0^t slope(\tau) \, d\tau$$

### 时间戳对齐与NTP同步

该机制牺牲了时间戳与实际采样时刻的绝对匹配，但换来了整齐的时间戳序列和更低的NTP访问频率。

系统通过双重机制确保数据时间戳的相对准确性与均匀性：

1. **NTP时间同步**：每次测量完成后（以及开机、断电恢复时），系统与NTP服务器校准时间并计算下一个10分钟整点作为下次采集时间，纠正内部RTC的累积漂移（嵌入式晶振误差可能导致每天数秒至数十秒的偏差），使数据时间戳尽可能符合实际时间
2. **整点对齐**：时间戳对齐到10分钟整点（`00:00:00`, `00:10:00`, `00:20:00`...）。单次测量流程耗时约45秒，但上传的时间戳是**计划采集时间**而非完成时间

**时间戳对齐的意义**：

| 潜在问题 | 未对齐时的影响 | 对齐机制的作用 |
| -------- | -------------- | -------------- |
| RTC累积漂移 | 运行一周后时间戳可能偏差数十分钟，数据与真实事件（例如光照/黑暗操作）产生错位 | 每次采集后NTP校准，确保长期运行的时间精度 |
| 多设备数据融合 | 不同设备时间基准不一致，无法进行横向对比分析 | 统一时间基准，支持多源数据的直接叠加与对比 |
| 数据索引效率 | 时间戳分散（`14:07:23`, `14:17:41`...），检索与归档效率低 | 规整时间戳（`14:00:00`, `14:10:00`...），便于索引与可视化 |

### 气泵闭路采样

PWM 控制气泵进行闭路气路采样：

1. 气泵循环（可调时长和强度）
2. 停泵，等待气压稳定（可调时长）
3. 读取传感器数据
4. 实时上传服务器

### 自动启动运行

- 上电自动启动，无需人工干预
- 多 WiFi 热点支持，自动切换，提高容错
- 断电恢复后自动继续
- 时间戳自动对齐到整点

### 完整物联网数据流

```
传感器 采样 → HTTP 上传 → PHP 存储 → ECharts 可视化
```

| GPIO 17 | ZG09SR TX |

## 快速开始

### 1. 固件配置

```bash
cd firmware
cp src/config.h.example src/config.h
# 编辑 config.h，填入个人的需求配置
```

使用 PlatformIO 编译上传：

```bash
pio run -t upload
```

### 2. 服务器部署

```bash
cd web
cp config.php.example config.php
# 编辑 config.php，填入数据库信息
```

导入示例数据（包含表结构和测试数据）：

```bash
mysql -u your_user -p co2data < data/sample_data.sql
```

或手动创建空表：

```sql
CREATE TABLE co2_measurements (
  id INT AUTO_INCREMENT PRIMARY KEY,
  timestamp DATETIME NOT NULL,
  co2_ppm INT NOT NULL,
  temperature FLOAT DEFAULT NULL,  -- 预留，暂未使用
  humidity FLOAT DEFAULT NULL,     -- 预留，暂未使用
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  INDEX idx_timestamp (timestamp)
);
```

### 3. 使用漂移补偿

1. 浏览器打开 `index.html`
2. 选择时间范围查询数据
3. 在"漂移补偿"面板添加锚点时段
   - 选择完整的**一个**光暗周期的起止点作为锚点时段
   - 系统会对该时段进行线性回归，计算斜率作为漂移速率的近似中值
   - 多个锚点之间会进行插值，实现漂移速率的平滑过渡
4. 点击"应用漂移补偿"查看效果

## 配置参数

在 `config.h` 中可调整：

| 参数 | 默认值 | 说明 |
| ---- | ------ | ---- |
| PUMP_DUTY_MEASURE | 120 | 采样气泵 PWM 占空比 (0-255) |
| PUMP_DUTY_VENT | 150 | 通风气泵 PWM 占空比 |
| PUMP_TIME_MEASURE | 25 | 采样气泵运行时间（秒） |
| PUMP_TIME_VENT | 50 | 通风气泵运行时间（秒） |
| SENSOR_STABILIZE_TIME | 20 | 气压稳定等待时间（秒） |
| INTERVAL_SECONDS | 600 | 测量间隔（固定10分钟，对齐整点）* |

> *注意：修改 `INTERVAL_SECONDS` 需同时修改对齐逻辑，参见源码中 `getNextAlignedEpoch()` 函数。单次完整测量耗时应远小于测量间隔，否则可能导致周期重叠。
